## **DERP 服务器租赁与管理系统 需求分析**

### **1. 用户管理模块**

#### 1.1 管理员角色与权限
- **超级管理员**：具有系统配置、权限分配与管理、分销系统设置等超级root权限，可以访问所有模块。
- **普通管理员**：只能查看和管理普通用户的业务流程，执行日常的更新、删除和维护操作，无法访问系统配置和财务系统。
- **分销管理员**：负责生成和管理序列号，并根据分销员的等级（黄金、白金、铂金）享受不同的折扣，管理财务分成。
- **普通用户**：主要负责租赁服务，使用服务器资源和容器，没有管理权限。

#### 1.2 管理员帮助用户续费
- 管理员可以帮助用户续费并指定续费结束日期，系统会更新用户服务到期时间，并发送续费成功的通知邮件。

#### 1.3 管理员注销用户
- **注销操作**：
  1. 检查用户的财务结算是否完整（包括续费等费用）。
  2. 销毁该用户所有的 ACL 文件。
  3. 释放与用户相关联的 Docker 容器资源（通过 SSH 登录）。
  4. 发送注销通知邮件给用户。

#### 1.4 用户注销功能
- 提前 7 天、3 天、1 天发送续费通知邮件，未续费的用户将关闭其 DERP 服务并保留用户权限。超过 30 天未续费的用户将自动注销。

#### 1.5 邮件发送频率限制
- 每隔 10 分钟，用户注册邮箱最多发送 5 封邮件。

---

### **2. 分销模块**

#### 2.1 分销管理员权限
- **分销管理员**：
  - 生成序列号，用以开辟服务器资源和权限。
  - 管理生成的序列号，包括与服务器、容器及 ACL 文件的关联。
  - 分销员按等级（黄金、白金、铂金）享受不同的折扣，管理财务分成。

#### 2.2 管理员查看分销序列号与用户关联
- **管理员权限**：
  - 可查看由其权限生成的序列号，并查看这些序列号所关联的普通用户情况。

#### 2.3 分销序列号生成与使用
- **序列号生成**：分销管理员生成带有时效性的序列号，并与服务器资源和容器关联。
- **序列号使用**：用户激活序列号后，系统自动为其分配相应的服务器资源和 Docker 容器。

---

### **3. 服务器管理模块**

#### 3.1 服务器信息管理
- **服务器列表**：管理员能够查看所有服务器的基本信息，监控其运行状态，并能根据需要进行相关管理操作。每台服务器展示的字段包括：
  - **阿里云账号（手机号）**、**IP 所在国内地址**、**公网 IP 地址**、**用户数（Docker 容器数量）**、**总体流量**、**月度剩余流量**。

#### 3.2 速率检测与 Ping 延时检测
- **速率检测**：管理员能够对服务器进行速率检测，确保其带宽稳定。
- **Ping 延时检测**：提供延时检测功能，判断服务器与客户端之间的连接稳定性。
- **监控报警**：检测到速率异常或延时过高时，系统能够发出报警通知，帮助管理员及时发现并处理问题。

#### 3.3 服务器资源管理
- **资源配额管理**：管理员能够管理每台服务器的资源配额，包括流量、Docker 容器数量、用户数等。
- **自动关闭超流量的 Docker 容器**：如果服务器的流量超过限制，系统应自动关闭 Docker 容器，以防服务器过载。
- **健康状态检测**：支持自动检测服务器的健康状态，如负载、响应时间、硬盘使用情况等。

#### 3.4 服务器扩展功能
- **新增服务器**：管理员可以添加新的服务器，并配置相应的资源。
- **删除服务器**：管理员可以删除不再需要的服务器资源，并进行资源回收。
- **服务器迁移**：支持将负载较高的 Docker 容器迁移至其他服务器，确保负载均衡。

---

### **4. Docker 容器管理模块**

#### 4.1 容器隔离
- 每个用户分配独立 Docker 容器和端口，支持动态分配端口，避免冲突。

#### 4.2 容器生命周期管理
- 到期后自动释放用户的容器。新续费时延长容器使用时间或重新分配。

#### 4.3 流量统计
- 按容器记录用户流量，包括实时速率和总流量。

---

### **5. 财务管理模块**

#### 5.1 套餐定价与序列号管理
- 管理员设置租赁套餐（3天、5天、30天、180天、360天），并自动生成复杂序列号供用户激活租赁。

#### 5.2 财务统计
- 显示当月收益、历史总收益，统计用户预付款余额和套餐购买数量。

#### 5.3 订单管理
- 查看用户订单历史，支持完成、续费和取消状态标记。

---

### **6. 系统安全性与日志管理**

#### 6.1 设备绑定
- ACL 配置仅允许用户注册设备访问。

#### 6.2 流量异常监控
- 检测用户流量异常情况（速率过高或超限）。

#### 6.3 权限与日志
- 管理员界面支持多级权限分配，并提供系统操作日志，追踪管理员和用户的关键操作。

### **7. 系统架构设计与实施步骤**

#### 7.1 系统结构
- **前端**：用户和管理员界面基于 React，支持动态展示用户和服务器状态。
- **后端**：RESTful API使用 Flask 或 FastAPI实现核心逻辑，异步任务队列使用 Celery 结合 Redis 处理容器释放、ACL 到期等任务。
- **数据库**：初期使用 SQLite，后期迁移到 MySQL 或 PostgreSQL。

#### 7.2 关键技术
- **自动化**：使用 Docker Compose 或 Kubernetes 管理服务，支持快速部署与扩展。
- **安全性**：引入多因素身份验证（MFA），动态 ACL 配置绑定设备和有效期。
- **高可用**：动态分配低负载服务器，提高服务稳定性。

#### 7.3 部署与实施步骤
1. **开发阶段**：开发核心功能，模块化开发，确保后期扩展和维护。
2. **初始部署**：使用云服务器部署系统，安装 Docker 和所需依赖。
3. **生产部署**：引入负载均衡和 Redis 缓存，部署 Prometheus + Grafana 监控系统，配置报警机制。

#### 7.4 后续扩展与优化
- **高可用架构**：使用 Kubernetes 或 Docker Swarm 实现多服务器集群管理。
- **财务功能扩展**：引入优惠券机制，支持促销活动。

---

### **8. 总结**
- **最优设计**：系统为模块化设计，具备高度的可扩展性，重点关注安全性、性能和用户体验。
- **逐步部署**：从单机版本开始，逐步扩展到多服务器集群，保障系统稳定性和高可用性。


### **1. 阶段性流程图设计**

#### **阶段 1：用户注册与登录流程**
- **用户注册**：用户输入邮箱进行注册，系统进行邮箱验证，成功后生成用户账户。
- **用户登录**：用户输入注册邮箱和密码，进行身份验证，通过后登录。
- **序列号激活**：用户通过管理员生成的序列号激活租赁服务。

#### **阶段 2：ACL 配置生成与管理**
- **ACL 生成**：用户租赁服务后，系统为用户生成三台服务器的 ACL 配置并绑定到注册设备。
- **ACL 更新与失效**：ACL 配置有效期为32天，到期后自动更新或管理员手动更新。
- **ACL 校验**：下载 ACL 配置时，系统进行权限校验，确保序列号已支付。

#### **阶段 3：服务器管理**
- **服务器列表显示**：管理员可查看所有服务器的基本信息，包括IP地址、流量使用情况、服务器状态等。
- **服务器资源管理**：根据流量和负载情况，管理员可自动关闭超流量的 Docker 容器，并进行健康监控。
- **服务器扩展功能**：管理员可以新增、删除服务器，并进行负载迁移。

#### **阶段 4：分销系统管理**
- **分销员操作**：分销管理员可生成和管理带有时间长度的序列号，绑定到用户租赁服务。
- **分销员财务管理**：根据分销员的等级，计算分销收益并进行财务结算。

#### **阶段 5：财务管理**
- **套餐定价与序列号管理**：管理员设置租赁套餐，并生成对应的序列号供用户激活。
- **财务统计与订单管理**：系统自动统计月度收益、套餐销售数量，并生成财务报告。
